## Screening Question Set（給抽取型 agent；請只提供本段，不要提供上面的 criteria）

> 目標：只做**資訊抽取 + 定位**，不要做是否納入/排除的結論。
> 回答格式建議：每題都用「(a) 結論：YES/NO/未明說」「(b) 原文 quote」「(c) 位置（Abstract/Section/Page/Table/Appendix）」。

### Q1. 研究主題/任務（Requirement Formalisation）
1. 摘錄 paper 中描述「requirements formalisation / formalising requirements / requirement formalization」或等價概念的一句原文（優先 Abstract / Introduction）。
2. 這篇 paper 的 requirements 指的是什麼情境（例如 software requirements / system requirements / stakeholder requirements…）？請摘錄最能支持的一句原文。
3. paper 是否明確把研究問題界定為「把自然語言需求轉成某種 formal model / representation / specification」？  
   - 若有：摘錄原文並寫出 formalisation 產物名稱（例如 logic formula / UML / goal model / state machine / ontology / DSL…）。

### Q2. 產出是否為「formalisation」（可被視為可篩選條件的核心訊號）
1. paper 的方法最後產出什麼（model / specification / representation / graph / template…）？請摘錄一段最能代表輸出定義的原文。
2. paper 是否聲稱「generate / produce / derive / extract / translate」某種 formal artefact from requirement(s)？  
   - 若有：摘錄原文；若無：回答「未明說」。

### Q3. 研究類型（Primary vs Survey/Review/Overview）
1. paper 是否自稱為 survey / review / systematic review / mapping study / tutorial / overview？  
   - 若有：摘錄原文（通常在 Title / Abstract / Introduction）。
2. paper 是否呈現「提出方法 + 實驗/案例驗證」的結構？  
   - 請列出能支持的章節名稱（例如 Method / Experiments / Case Study / Evaluation）與對應的原文片段（可多段）。

### Q4. 出版資訊（Venue / Peer-review / 年份 / 語言）——**可由 metadata 程式處理，但仍請抽取可見證據**
1. 摘錄 paper 的發表年份/日期（若 paper 內有明示，例如封面或 footnote）。沒有就回答「以 metadata 為準」。
2. paper 的發表型態是什麼？（journal / conference / workshop / arXiv preprint / technical report / white paper / book chapter …）  
   - 請摘錄最能支持的原文（例如 “Proceedings of …”, “Journal of …”, “arXiv preprint”）。
3. paper 是否明確標示語言為 English 或全文為英文？  
   - 若 paper 內未明示：回答「未明說」（但可補充：全文觀察為英文/非英文）。

### Q5. 非期刊/非會議/非工作坊類型排除訊號（只抽取，不下結論）
> 這題只是在找「可能是書籍/網站/技術報告/白皮書/小冊子/教學文」等訊號。
1. 若 paper 內出現以下任一詞彙，請逐一摘錄與定位：  
   - “technical report”, “white paper”, “tutorial”, “pamphlet”, “book”, “chapter”, “website”
2. 若看起來是「書籍章節」或「白皮書」但 paper 內沒有明示：回答「未明說」，不要自行判定。

### Q6. 重複/版本識別（用於後續去重；只抽取）
1. 列出所有可用的唯一識別：DOI / arXiv ID / ACL Anthology ID / IEEE/ACM article number（有多少列多少）。
2. 若同一篇 paper 看起來有 multiple versions（preprint vs published）：摘錄能支持版本關係的一句原文或 footnote（若無則答未明說）。

### Q7. 品質評估訊號（對應 QA1–QA5；只抽取 evidence）
1. 研究是否清楚定義 research objectives？摘錄一段最能代表 objectives 的原文。
2. 是否清楚陳述 limitations / threats / future work？摘錄原文（可多段）。
3. 是否使用或建立 dataset？  
   - 若有：dataset 名稱是什麼？是否 publicly available？請各自摘錄原文。
4. 是否與其他 SOTA technique 做比較？  
   - 若有：摘錄比較敘述或結果表格附近原文（Table/Section）。
5. 結果/發現是否有清楚陳述？  
   - 摘錄一段最能代表 main findings 的原文（通常在 Results / Conclusion）。
