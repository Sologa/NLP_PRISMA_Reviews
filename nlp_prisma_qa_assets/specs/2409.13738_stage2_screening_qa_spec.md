## 設計原則

- 這份 spec 只對齊 **current active** `criteria_stage2/2409.13738.json`。
- Stage 2 extraction only; final include/exclude is decided downstream from the extracted evidence object plus metadata filter.
- 問題必須只做資訊抽取與定位，不直接下 include/exclude verdict。
- 每個回答都要有 quote 與 location；找不到時要明寫 `未明說`。
- 不得把歷史 criteria drift、retrieval gate、或 performance-oriented hardening 偷渡成 current criteria。

## Metadata-only / operational notes

- Publication form, venue type, peer-review status, full-text availability, and language belong to the separate metadata_filter_spec for 2409.
- This QA spec only asks for visible in-paper evidence when available; metadata remains the authoritative source for those fields.
- Do not add new operational hardening beyond the current canonical Stage 2 criteria.

## 本 spec 期望產出的 evidence fields

- `paper_type`: 是否為 full research article，而非 poster / short paper / abstract-only / report / thesis / book chapter。
- `peer_reviewed_or_venue_type`: conference / journal / preprint / workshop / other；最終以 metadata 判定。
- `fulltext_availability`: 是否可取得全文；最終以 metadata 判定。
- `language`: 是否為 English；最終以 metadata 判定。
- `primary_research_signal`: 是否呈現 original research contribution，而非 secondary research。
- `task_sentence`: 最能代表 paper 任務的一句原文。
- `source_object`: 來源是否為 natural-language text。
- `target_object`: 目標是否為 process representation / model。
- `nlp_role`: NLP 是否為核心 extraction 方法。
- `concrete_method_signal`: 是否有具體方法/pipeline/model 描述。
- `experimental_validation_signal`: 是否有 experiments / evaluation / empirical validation。
- `non_target_task_signal`: 是否出現 redesign / matching / prediction / sentiment / label-only / text-from-process generation 等反證。
- `secondary_research_signal`: 是否為次級研究。
- `conflict_note`: 正反證據互相衝突時的備註。

## 單題回答格式

每個問題至少輸出以下欄位：
- `field_name`
- `answer`
- `state = {present, absent, unclear, not_observable}`
- `evidence_quotes[]`
- `location = {abstract, introduction, method, experiments, conclusion, front matter, appendix, table/figure}`
- `missingness_reason = {not_stated, not_found, conflicting, metadata_only, stage1_not_observable}`
- `conflict_note`

建議規則：
- `state=present`：找到直接支持該欄位的證據
- `state=absent`：找到直接反證，或原文明示相反
- `state=unclear`：有相關線索但不足以下判
- `state=not_observable`：只在 Stage 2 的 full text / metadata 才能判，或此階段原則上不該硬判

## Screening Question Set

### Q0. 可見 metadata 證據（非最終決策來源）

若 full text front matter 有 bibliographic line、venue 名稱、article type、language、open-access / publisher note，請逐字摘錄。若沒有，回答「未明說」。注意：這些欄位最終仍以 metadata_filter_spec 為準。

- 對應 evidence fields: `paper_type`, `peer_reviewed_or_venue_type`, `fulltext_availability`, `language`
- 對應 criteria refs: `I1`, `E1`

### Q1. Primary research vs secondary research

摘錄能支持 original research contribution 的原文（例如 we propose/present/develop）與任何 review/survey/mapping/meta-analysis 訊號。若兩邊都看得到，兩邊都要列出並寫入 conflict_note。

- 對應 evidence fields: `primary_research_signal`, `secondary_research_signal`, `conflict_note`
- 對應 criteria refs: `I2`, `E2`

### Q2. 任務主句與 topic fit

摘錄 1–2 句最能代表 paper 是否 specifically covers NLP for process extraction from natural-language text 的原文。優先使用 abstract / intro / problem definition 中的句子。

- 對應 evidence fields: `task_sentence`, `nlp_role`
- 對應 criteria refs: `I3`, `E3`

### Q3. 來源物件與目標物件

分別摘錄 source object（requirements/manuals/emails/policies/free text 等）與 target object（process model / process representation / BPMN / Petri net / control-flow / decisional model 等）的原文。若 target 其實是 prediction、matching、labels、sentiment 或 text generation，也要如實摘錄。

- 對應 evidence fields: `source_object`, `target_object`, `non_target_task_signal`
- 對應 criteria refs: `I3`, `E3`

### Q4. 具體方法（concrete method）

摘錄最能代表具體方法的段落或句子，至少回答 paper 是否描述 algorithm / pipeline / model / architecture / procedure。若只有概念性討論沒有方法細節，摘錄支持原文。

- 對應 evidence fields: `concrete_method_signal`
- 對應 criteria refs: `I4`, `E4`

### Q5. 實驗與實證驗證

摘錄 experiments / evaluation / empirical study 的直接證據。至少提供一段含 dataset、metrics、baselines、results、或 experiment section/table 的原文；若沒有，回答「未明說」並說明沒找到哪一類證據。

- 對應 evidence fields: `experimental_validation_signal`
- 對應 criteria refs: `I4`, `E4`

### Q6. 非目標家族反證

尋找並摘錄任何顯示 paper 其實在做 process redesign、matching、prediction、sentiment analysis、label-only target、或從 process 生成自然語言的原文。若沒有明說，回答「未明說」。

- 對應 evidence fields: `non_target_task_signal`
- 對應 criteria refs: `E3`

### Q7. 衝突與 unresolved 欄位

若不同段落對 task fit、method concreteness、或 empirical validation 給出互相牴觸的訊號，請整理成 conflict_note；若仍有不能定的欄位，列出 unresolved fields 與缺失原因。

- 對應 evidence fields: `conflict_note`
- 對應 criteria refs: `I3`, `I4`, `E3`, `E4`

## Criteria coverage matrix

| criteria ref | criterion text | covered by questions |
|---|---|---|
| `I1` | Paper is a full research article published in a peer-reviewed conference or journal, a full text is available, and the paper is written in English. | `Q0` |
| `I2` | Paper is a primary research article presenting an original research contribution. | `Q1` |
| `I3` | Paper specifically covers the use of NLP for process extraction from natural language text. | `Q2`, `Q3`, `Q7` |
| `I4` | Paper explicates a concrete method and describes experiments to empirically validate the method. | `Q4`, `Q5`, `Q7` |
| `E1` | Exclude papers that are not full research papers, not in a conference or journal, not peer-reviewed including preprints, without full text, or not in English. | `Q0` |
| `E2` | Exclude secondary research articles. | `Q1` |
| `E3` | Exclude papers not specifically covering NLP for process extraction, including redesign, matching, prediction, sentiment, label-only, or text-from-process generation. | `Q2`, `Q3`, `Q6`, `Q7` |
| `E4` | Exclude papers that only superficially discuss the method or do not describe experiments. | `Q4`, `Q5`, `Q7` |

## Handoff guidance

- Stage 1 → Stage 2：只把 unresolved / conflicting / boundary-sensitive 欄位往下帶，不要丟失已抽到的正反證。
- Stage 2 → SeniorLead：優先交付 synthesized evidence object；只有必要時才展開 raw quotes。
- 若同一欄位存在正反證據，請不要自行合併成結論；改寫入 `conflict_note`。
