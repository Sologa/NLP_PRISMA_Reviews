## 設計原則

- 這份 spec 只對齊 **current active** `criteria_stage2/2511.13936.json`。
- Stage 2 extraction only; final include/exclude is decided downstream from the extracted evidence object.
- 問題必須只做資訊抽取與定位，不直接下 include/exclude verdict。
- 每個回答都要有 quote 與 location；找不到時要明寫 `未明說`。
- 不得把歷史 criteria drift、retrieval gate、或 performance-oriented hardening 偷渡成 current criteria。

## Metadata-only / operational notes

- Current active Stage 2 criteria contain no mandatory metadata-only inclusion gate.
- Do not import historical retrieval constraints (2020+, arXiv citation cutoffs, venue filters) into current Stage 2 screening logic.
- Descriptive metadata may still be recorded for provenance, but it is not an eligibility rule in this spec.

## 本 spec 期望產出的 evidence fields

- `task_sentence`: 最能代表 paper 任務的一句原文。
- `preference_learning_signal`: 是否有符合 current canonical 定義的 preference learning。
- `comparison_type`: ranking / A-B / pairwise / none / unclear。
- `rating_to_preference_conversion`: numeric ratings 是否明確轉成 preference / ranking。
- `preference_signal_source`: human / synthetic / mixed / unclear。
- `rl_loop_for_audio_model`: 無 explicit comparison 時是否存在 RL training loop for an audio model。
- `optimization_target`: preference / reward 訊號作用在哪個 model / policy / decoder / generator。
- `reward_model_or_preference_model`: 是否存在 reward model / preference model / scorer。
- `learning_vs_evaluation_role`: preference 主要用於 learning 還是僅用於 evaluation。
- `audio_domain_signal`: paper 是否確實位於 audio domain。
- `multimodal_with_audio`: multimodal work 是否包含 audio。
- `survey_signal`: 是否為 survey / review。
- `conflict_note`: learning-role、audio-boundary、或 evaluation-only 的衝突備註。

## 單題回答格式

每個問題至少輸出以下欄位：
- `field_name`
- `answer`
- `state = {present, absent, unclear, not_observable}`
- `evidence_quotes[]`
- `location = {abstract, intro, method, training, experiments, appendix, table/figure}`
- `missingness_reason = {not_stated, not_found, conflicting, stage1_not_observable}`
- `conflict_note`

建議規則：
- `state=present`：找到直接支持該欄位的證據
- `state=absent`：找到直接反證，或原文明示相反
- `state=unclear`：有相關線索但不足以下判
- `state=not_observable`：只在 Stage 2 的 full text / metadata 才能判，或此階段原則上不該硬判

## Screening Question Set

### Q1. Core preference-learning evidence

摘錄最能支持 paper pertain to preference learning 的原文。優先找 ranking、A/B comparison、pairwise preference、preference optimization 等直接描述。若 paper 只說 user ratings / reward / feedback，請把原句完整摘錄。

- 對應 evidence fields: `preference_learning_signal`, `comparison_type`, `task_sentence`
- 對應 criteria refs: `I1`, `E1`

### Q2. Preference signal source

摘錄 preference 訊號來自 human judgments、synthetic preferences、還是 mixed 的原文。若 paper 沒有說明來源，回答「未明說」。

- 對應 evidence fields: `preference_signal_source`
- 對應 criteria refs: `I1`

### Q3. Numeric ratings → ranking / pairwise conversion

若 paper 使用 MOS、Likert、quality scores 或其他 numeric ratings，摘錄是否明確把這些 ratings 轉成 ranking / pairwise preferences 的原文；若沒有轉換，直接寫出反證。

- 對應 evidence fields: `rating_to_preference_conversion`
- 對應 criteria refs: `I1`

### Q4. RL loop、optimization target、reward/preference model

摘錄任何 RL training loop、reward model、preference model、policy optimization、或 preference-driven fine-tuning 的原文，並指出 preference signal 作用的對象是什麼（audio model / policy / generator / scorer）。若沒有這些元件，也要寫明。

- 對應 evidence fields: `rl_loop_for_audio_model`, `optimization_target`, `reward_model_or_preference_model`
- 對應 criteria refs: `I1`

### Q5. Learning vs evaluation role

摘錄能判斷 preference signal 是用在 training/optimization/fine-tuning 還是只用在 evaluation 的原文。若兩種角色都出現，兩邊都要列出並寫入 conflict_note。

- 對應 evidence fields: `learning_vs_evaluation_role`, `conflict_note`
- 對應 criteria refs: `I1`, `E1`

### Q6. Audio-domain 與 multimodal 邊界

摘錄 paper 落在 audio / speech / music / audio-text domain 的原文。若為 multimodal，請再摘錄 audio 與其他模態並存的原文，避免只憑模糊的 'multimodal' 字樣推論。

- 對應 evidence fields: `audio_domain_signal`, `multimodal_with_audio`
- 對應 criteria refs: `I2`

### Q7. Survey / review 排除

摘錄任何 survey / review / systematic review / overview / tutorial / taxonomy 等訊號。若沒有，仍需提供一段支持它是 original technical paper 的方法或實驗原文。

- 對應 evidence fields: `survey_signal`
- 對應 criteria refs: `I3`, `E2`

### Q8. 衝突與 unresolved 欄位

若 preference-learning evidence、learning-vs-evaluation、或 audio-domain evidence 互相衝突，請整理成 conflict_note；若仍有無法判定的欄位，列出 unresolved fields 與缺失原因。

- 對應 evidence fields: `conflict_note`
- 對應 criteria refs: `I1`, `I2`, `I3`, `E1`, `E2`
