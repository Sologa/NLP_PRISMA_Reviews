## 本 spec 期望產出的 evidence fields

- `task_sentence`: 最能代表 paper 任務的一句原文。
- `preference_learning_signal`: 是否存在 ranking / A-B / pairwise preference / preference learning 的核心訊號。
- `comparison_type`: ranking / A-B / pairwise / none / unclear。
- `rating_to_preference_conversion`: numeric ratings（如 MOS）是否明確轉成 ranking / pairwise preference。
- `rl_loop_for_audio_model`: 沒有 explicit comparison 時，是否仍存在 RL training loop for an audio model。
- `learning_vs_evaluation_role`: preference 訊號是用於 learning / optimization / fine-tuning，還是只用於 evaluation。
- `audio_domain_signal`: 是否真的落在 audio / speech / music / audio-text domain。
- `multimodal_with_audio`: 若 paper 是 multimodal，audio 是否明確出現且是被納入的模態。
- `survey_signal`: 是否為 survey / review / overview / meta-analysis 等次級研究。
- `stage1_uncertainty`: 需要 Stage 2 補看的邊界點。

## Screening Question Set

### Q1. 任務主句

從 title 或 abstract 摘錄 1 句最能代表 paper 主任務的原文。不要改寫。若 abstract 同時有方法目標與評估目標，兩句都列出。

- 對應 evidence fields: `task_sentence`
- 對應 criteria refs: `I1`, `I4`

### Q2. Preference-learning 核心證據

摘錄任何 ranking、A/B comparison、pairwise preference、preference learning、comparison of audio clips 等原文。若只看到 general reward / feedback，而看不到偏好比較，也要如實摘錄。

- 對應 evidence fields: `preference_learning_signal`, `comparison_type`
- 對應 criteria refs: `I1`, `E1`

### Q3. Numeric ratings 是否轉成 preference

若 abstract 提到 MOS、ratings、scores、Likert 等 numeric ratings，請摘錄是否明示把 ratings 轉成 ranking / pairwise preference 的原文。若沒有這種轉換證據，回答「未明說」。

- 對應 evidence fields: `rating_to_preference_conversion`
- 對應 criteria refs: `I2`

### Q4. RL training loop / preference-driven optimization

若沒有明確 ranking/A-B/pairwise evidence，請摘錄是否存在 RL training loop、preference-driven optimization、reward model、policy optimization、RLHF/DPO/KTO 類訊號，而且目標是 audio model。若都沒有，回答「未明說」。

- 對應 evidence fields: `rl_loop_for_audio_model`
- 對應 criteria refs: `I3`

### Q5. Preference 用於 learning 還是 evaluation

摘錄最能顯示 preference signal 被用在 training / optimization / fine-tuning / learning，或只用在 evaluation / assessment 的原文。若兩者都有跡象，兩邊都要列出。

- 對應 evidence fields: `learning_vs_evaluation_role`
- 對應 criteria refs: `I1`, `I3`, `E1`

### Q6. Audio-domain 證據

摘錄能證明 paper 位於 audio / speech / music / audio-text domain 的原文。若 paper 看起來是跨模態，仍要先把 audio 相關 evidence 單獨摘錄。

- 對應 evidence fields: `audio_domain_signal`
- 對應 criteria refs: `I4`, `E2`

### Q7. Multimodal with audio

若 abstract 顯示為 multimodal，摘錄 audio 與其他模態同時出現的原文；若不是 multimodal，回答「not_applicable」並說明為何。

- 對應 evidence fields: `multimodal_with_audio`
- 對應 criteria refs: `I4`

### Q8. Survey / review 訊號

摘錄任何 survey / review / systematic review / overview / tutorial / meta-analysis / benchmark-overview 類訊號。若 title/abstract 沒有明說，也要寫「未明說」。

- 對應 evidence fields: `survey_signal`
- 對應 criteria refs: `I5`, `E3`

### Q9. Stage 1 handoff 不確定點

若 core fit plausible 但 comparison type、rating conversion、RL loop、learning-vs-evaluation、或 audio-domain evidence 仍不足，請列出 unresolved 欄位與 Stage 2 要補看的位置。不要做 include/exclude 結論。

- 對應 evidence fields: `stage1_uncertainty`
- 對應 criteria refs: `I1`, `I2`, `I3`, `I4`, `E1`
