## 設計原則

- 這份 spec 只對齊 **current active** `criteria_stage1/2409.13738.json`。
- Stage 1.2 extraction only; no include/exclude decision in this spec.
- 問題必須只做資訊抽取與定位，不直接下 include/exclude verdict。
- 每個回答都要有 quote 與 location；找不到時要明寫 `未明說`。
- 不得把歷史 criteria drift、retrieval gate、或 performance-oriented hardening 偷渡成 current criteria。

## Metadata-only / operational notes

- Current active Stage 1 criteria do not define any mandatory metadata-only decision gate.
- Publication form, peer review, full-text availability, and language are explicitly deferred to Stage 2.
- You may record descriptive metadata (year, venue string, visible language cues), but these fields must not be used as Stage 1 hard exclusions in this spec.

## 本 spec 期望產出的 evidence fields

- `task_sentence`: 最能代表 paper 任務的一句原文。
- `source_object`: paper 的輸入/來源物件是否為 natural-language text；若只看見 labels / logs / structured fields，也要如實摘錄。
- `target_object`: paper 的輸出/目標物件是否為 process representation / model；若其實是 prediction / matching / labels / text generation，也要如實摘錄。
- `nlp_role`: NLP 是核心 extraction 方法，還是只是相鄰背景技術。
- `non_target_task_signal`: 是否出現 process redesign / matching / process prediction / sentiment analysis / label-only / text-from-process generation 等反證。
- `secondary_research_signal`: 是否有 survey / review / mapping / meta-analysis / overview / tutorial / SoK 等次級研究訊號。
- `stage1_uncertainty`: 若核心 fit plausible 但證據不足，需明確列出哪些欄位要 defer 到 Stage 2。

## 單題回答格式

每個問題至少輸出以下欄位：
- `field_name`
- `answer`
- `state = {present, absent, unclear}`
- `evidence_quotes[]`
- `location = {title, abstract}`
- `missingness_reason = {not_stated, not_found, conflicting}`
- `handoff_note`

建議規則：
- `state=present`：找到直接支持該欄位的證據
- `state=absent`：找到直接反證，或原文明示相反
- `state=unclear`：有相關線索但不足以下判
- `state=not_observable`：只在 Stage 2 的 full text / metadata 才能判，或此階段原則上不該硬判

## Screening Question Set

### Q1. 任務主句（task sentence）

從 title 或 abstract 摘錄 1 句最能代表 paper 主任務的原文。不要改寫。若 title 與 abstract 的任務描述不一致，兩句都要列出。

- 對應 evidence fields: `task_sentence`
- 對應 criteria refs: `R1`, `R2`

### Q2. 來源物件是否為 natural-language text

摘錄能回答「input / source object 是什麼」的原文。若 paper 只提到 labels、structured records、event logs、tables、signals，而沒有自然語言文字作為來源物件，也要逐字摘錄。

- 對應 evidence fields: `source_object`
- 對應 criteria refs: `R1`, `R2`, `E3`

### Q3. 目標物件是否為 process representation / model

摘錄能回答「paper 想抽取/建構的是什麼」的原文。請特別注意 process model、process representation、BPMN、Petri net、activity sequence、control-flow、decisional model 等正向訊號；若其實是 prediction、matching、redesign、labels、sentiment、或從 process 生成文字，也要原文摘錄。

- 對應 evidence fields: `target_object`, `non_target_task_signal`
- 對應 criteria refs: `R2`, `E2`, `E3`

### Q4. NLP 的角色

摘錄最能顯示 NLP 在 paper 裡扮演什麼角色的原文。若 NLP 是核心 extraction pipeline / model / method，摘錄支持句；若 NLP 只是背景、資料前處理、或 topic-adjacent，也要摘錄支持句。

- 對應 evidence fields: `nlp_role`
- 對應 criteria refs: `R1`, `R2`, `E1`

### Q5. 非目標家族反證

在 title/abstract 中尋找下列反證並摘錄原文：process redesign、process matching、process prediction、sentiment analysis、targeting individual labels instead of natural text、generating natural text from processes。若都沒明說，回答「未明說」。

- 對應 evidence fields: `non_target_task_signal`
- 對應 criteria refs: `E2`, `E3`

### Q6. 是否為次級研究

摘錄任何 survey / review / systematic review / mapping study / meta-analysis / overview / tutorial / roadmap / SoK 等訊號。若 title/abstract 未明示，也要明確寫「未明說」。

- 對應 evidence fields: `secondary_research_signal`
- 對應 criteria refs: `E4`

### Q7. Stage 1 handoff 不確定點

若目前 evidence 顯示 paper 可能符合主題，但 source object、target object、NLP role、或 non-target counterevidence 仍不足以判定，請列出 unresolved 欄位與需要在 Stage 2 補看的資訊。不要做 include/exclude 結論。

- 對應 evidence fields: `stage1_uncertainty`
- 對應 criteria refs: `R3`

## Handoff guidance

- Stage 1 → Stage 2：只把 unresolved / conflicting / boundary-sensitive 欄位往下帶，不要丟失已抽到的正反證。
- Stage 2 → SeniorLead：優先交付 synthesized evidence object；只有必要時才展開 raw quotes。
- 若同一欄位存在正反證據，請不要自行合併成結論；改寫入 `conflict_note`。
