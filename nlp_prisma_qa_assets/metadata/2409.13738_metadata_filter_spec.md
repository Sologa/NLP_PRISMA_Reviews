# 2409.13738 metadata_filter_spec

scope: Current active criteria alignment for 2409 stage-split workflow
authority: criteria_stage2/2409.13738.json is canonical; Stage 1 defers metadata-like checks to Stage 2.

## Rule table

| field | requirement | applies_to | decision_role | notes |
|---|---|---|---|---|
| `venue_type` | conference or journal | Stage 2 | `hard gate` | Use authoritative metadata; if only in-paper evidence is available, keep as observational until metadata confirms. |
| `is_peer_reviewed` | true | Stage 2 | `hard gate` | Preprints fail this check. |
| `publication_form` | full research article; exclude poster / short paper / abstract-only / report / thesis / book chapter | Stage 2 | `hard gate` | May require combined venue metadata + front-matter evidence. |
| `full_text_available` | true | Stage 2 | `hard gate` | Stage 1 does not use this as an exclusion. |
| `paper_language` | English | Stage 2 | `hard gate` | Stage 1 defers language confirmation. |
| `publication_year` | no restriction | Stage 1 + Stage 2 | `non-gate` | Current active criteria explicitly do not impose a year filter. |

## Explicit non-rules

- Do not import any historical criteria_jsons hardening.
- Do not add citation-based, arXiv-based, or extra venue prestige filters.
- Do not convert Stage 2 metadata gates into Stage 1 hard exclusions.

## Operational note

這份 metadata spec 的目的只有兩個：

1. 把可以程式化的欄位與真正需要閱讀內容的欄位拆開。
2. 防止 historical retrieval logic、citation gate、或 convenience heuristics 被誤當 current screening criteria。
