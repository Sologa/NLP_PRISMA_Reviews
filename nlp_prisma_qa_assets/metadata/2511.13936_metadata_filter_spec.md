# 2511.13936 metadata_filter_spec

scope: Current active criteria alignment for 2511 stage-split workflow
authority: criteria_stage1/2511.13936.json and criteria_stage2/2511.13936.json contain no mandatory metadata-only gate.

## Rule table

| field | requirement | applies_to | decision_role | notes |
|---|---|---|---|---|
| `publication_year` | record only; not an active screening gate | Stage 1 + Stage 2 | `descriptive_only` | The historical 2020+ retrieval window in criteria_mds must not be imported into current screening. |
| `venue_or_source` | record only; not an active screening gate | Stage 1 + Stage 2 | `descriptive_only` | Venue prestige / source filters are historical retrieval logic, not current eligibility logic. |
| `is_arxiv` | record only; not an active screening gate | Stage 1 + Stage 2 | `descriptive_only` | Do not apply arXiv-only citation cutoffs in current screening. |
| `citation_count` | record only when available; not an active screening gate | Stage 1 + Stage 2 | `descriptive_only` | Historical Table A.2 citation gates belong to old retrieval logic only. |
| `full_text_available` | record if available for operational convenience; not a stated active criterion | Stage 2 | `descriptive_only` | Useful for workflow plumbing, but not a canonical 2511 eligibility rule. |
| `paper_language` | record if available; not a stated active criterion | Stage 1 + Stage 2 | `descriptive_only` | Do not harden current criteria by adding an English-only gate. |

## Explicit non-rules

- Do not import 2020+ / citation-gate / venue-filter retrieval constraints from criteria_mds into the active screening decision.
- Do not treat metadata descriptives as hidden hard exclusions.
- Keep all content-based eligibility in the QA / synthesis / decision layers, not in metadata.

## Operational note

這份 metadata spec 的目的只有兩個：

1. 把可以程式化的欄位與真正需要閱讀內容的欄位拆開。
2. 防止 historical retrieval logic、citation gate、或 convenience heuristics 被誤當 current screening criteria。
